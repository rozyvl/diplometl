# ETL-процесс для хранилища данных NYC Taxi Trips

## Описание проекта

Проект разработан в рамках дипломной работы по теме «Разработка ETL-процессов для хранилища данных организации».

Цель проекта — реализовать ETL-процесс для загрузки, очистки, проверки качества и размещения данных в хранилище данных.

В качестве источника используются открытые данные NYC Taxi Trips, а именно данные о поездках жёлтого такси за январь 2024 года.

## Используемые инструменты

- Apache Airflow — оркестрация ETL-процесса;
- Python — обработка данных;
- Pandas — чтение и подготовка данных;
- PostgreSQL — хранилище данных;
- DBeaver — просмотр и проверка данных;
- Docker Compose — запуск локального окружения.

## Архитектура проекта

ETL-процесс состоит из следующих этапов:

1. Загрузка исходного parquet-файла.
2. Размещение данных в staging-слое.
3. Проверка качества данных.
4. Очистка и преобразование данных.
5. Загрузка данных в DWH-слой.
6. Формирование аналитической витрины.
7. Финальная проверка результата.

Общая схема обработки:

```text
NYC Taxi Trips
      ↓
staging.stg_yellow_taxi_trips
      ↓
dwh.dim_date
dwh.dim_vendor
dwh.dim_payment_type
dwh.dim_location
dwh.fact_trips
      ↓
mart.v_daily_taxi_report
