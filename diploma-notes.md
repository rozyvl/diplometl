# Заметки для дипломной работы

## Глава 2. Проектирование решения

### Архитектура проекта
Проект реализуется как локальный ETL-процесс для загрузки, обработки и размещения данных в хранилище данных.

### Используемые инструменты
- Apache Airflow — оркестрация ETL-процесса.
- Python — обработка данных.
- PostgreSQL — хранилище данных.
- Docker — локальное развёртывание сервисов.
- DBeaver — просмотр таблиц и выполнение SQL-запросов.

## Глава 3. Реализация решения

- Структура проекта в VS Code.
- Запущенные контейнеры Docker.

- Подключение к PostgreSQL в DBeaver.
- Таблицы staging, dwh и mart.

### Исправление проверки диапазона дат

После первичного формирования витрины были обнаружены записи с датами, выходящими за пределы января 2024 года. В ETL-процесс была добавлена дополнительная проверка качества данных, ограничивающая допустимый диапазон дат периодом с 2024-01-01 по 2024-02-01.

После повторного запуска DAG обе задачи Airflow завершились успешно.
